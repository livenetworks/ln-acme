# Plan: v1.1 Phase 5 — Motion Safety + Easing Retrofit

**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 5
**Branch:** `v1.1/phase-5-motion-safe`
**Depends on:** Phase 1 (`motion-safe` mixin, named easings)
**Unblocks:** Nothing hard-blocked, but Phase 6 visuals benefit

---

## Context

ln-acme v1.0 plays every transition and keyframe animation
unconditionally — modal slide-in, toggle collapse, loader spin, badge
live pulse, nav underline spring — regardless of the user's
`prefers-reduced-motion: reduce` preference. This is an accessibility
gap, not a style gap.

This phase audits every `@keyframes` and every non-color `transition`
in the library, wraps animation-only declarations in `motion-safe {}`,
and replaces inline bezier curves with named easing tokens where the
name matches intent.

**Not in scope:** color transitions on `:hover` — those are fine and
don't trigger vestibular issues. Leave them alone.

## Pre-resolved ambiguities

### A. What counts as "motion that needs gating"

GATE (wrap in `motion-safe`):
- `@keyframes` animations (spin, pulse, fade, slide)
- `transform` transitions (translate, scale, rotate)
- `opacity` transitions that move content in/out of view
- `height` / `max-height` / `width` transitions that resize elements
- `top` / `left` / `transform: translate` positioning moves

DO NOT GATE:
- `color` / `background-color` / `border-color` transitions (on hover, focus, active)
- `box-shadow` transitions that only change color
- Any transition where the user would find it disorienting for hover
  to *not* have a color animation

### C. Easing replacement policy

Only replace a hardcoded bezier with a named easing when the named
easing is the **same curve or closer**. Never change curve shape
during this phase — it's a retrofit, not a redesign.

Named easings from Phase 1:

```
--easing-standard:   cubic-bezier(0.4, 0, 0.2, 1)    // matches --transition-*
--easing-decelerate: cubic-bezier(0, 0, 0.2, 1)      // enter
--easing-accelerate: cubic-bezier(0.4, 0, 1, 1)      // exit
--easing-spring:     cubic-bezier(0.34, 1.56, 0.64, 1)
```

If a file has `cubic-bezier(0.4, 0, 0.2, 1)` inline, replace with
`var(--easing-standard)`. If it has a spring curve, replace with
`var(--easing-spring)`. If it has anything else, leave it alone.

### D. Commit strategy

One commit per logical group:

1. `feat(motion): wrap keyframe animations in motion-safe`
2. `feat(motion): wrap transform/height transitions in motion-safe`
3. `refactor(motion): replace hardcoded easings with named tokens`

Three commits total. Each small and reviewable.

### Step 3 — Gate keyframe animations (Commit 1)

For each file identified as needing a `@keyframes` gate, wrap the
**usage site** of the animation, not the keyframes declaration
itself. Example — `scss/components/_loader.scss`:

**Before:**

```scss
.loader {
	animation: spin 1s linear infinite;
}

@keyframes spin {
	to { transform: rotate(360deg); }
}
```

**After:**

```scss
@use '../config/mixins' as *;

.loader {
	@include motion-safe {
		animation: spin 1s linear infinite;
	}
}

@keyframes spin {
	to { transform: rotate(360deg); }
}
```

The keyframes declaration is static CSS — it's cheap and harmless
when unused, so no gate needed. What matters is whether the
`animation:` property references it.

Apply the same pattern to every `@keyframes` usage site from the
audit.

**Critical:** ensure each file that adds a `@include motion-safe`
already has `@use '../config/mixins' as *;` (or equivalent) at the
top. If not, add the `@use` directive.

**Verify:**

```bash
npm run build
```

Open `demo/admin/loader.html` (or any page with a spinner). With
DevTools → Rendering → `prefers-reduced-motion: reduce` enabled, the
spinner should freeze. Disabled, it should spin.

**Commit 1:**

```bash

```

Use `-A` scoped to the directories, not the repo root — avoids

### Step 4 — Gate transform/height transitions (Commit 2)

For each `transition: transform` or `transition: height` or
`transition-property` including `transform` / `height`, wrap the
enclosing property block in `motion-safe`.

Example — `scss/config/mixins/_collapsible.scss` (hypothetical):

**Before:**

```scss
@mixin collapsible {
	height: 0;
	overflow: hidden;
	transition: height 0.3s ease;
}
```

**After:**

```scss
@use 'motion' as *;

@mixin collapsible {
	height: 0;
	overflow: hidden;

	@include motion-safe {
		transition: height 0.3s ease;
	}
}
```

Note: use `@use 'motion' as *;` with a relative path appropriate for
the file's location. Keep existing `@use` lines unchanged.

Apply this pattern to every candidate from the audit.

### Step 5 — Replace hardcoded easings (Commit 3)

Find and replace:

```
cubic-bezier(0.4, 0, 0.2, 1)  →  var(--easing-standard)
cubic-bezier(0, 0, 0.2, 1)    →  var(--easing-decelerate)

curves and any decorative springs outside the official set).

**Critical exception:** `_tokens.scss` `--transition-base/fast/slow`
keep their inline bezier — those values ARE the canonical source.

### Step 6 — Document

Create `docs/css/motion.md`:

```markdown
# Motion

Files:
- `scss/config/mixins/_motion.scss` — `motion-safe` mixin
- `scss/config/_tokens.scss` — duration and easing tokens

## Philosophy

ln-acme respects `prefers-reduced-motion: reduce`. Every animation
and every transform/height/opacity transition is gated so users who
have opted out of motion see instant state changes.

**What is gated:**
- `@keyframes` animations (spin, pulse, fade, slide)
- `transform` transitions (translate, scale, rotate)
- `opacity` transitions on appearance/disappearance
- `height` / `max-height` / `width` transitions

**What is NOT gated:**
- `color` transitions on hover/focus/active — these do not cause
  vestibular issues and removing them would make interactive
  elements feel broken.

## Using motion-safe

```scss
@use 'ln-acme/scss/config/mixins' as *;

.my-animated-panel {
	opacity: 0;
	transform: translateY(1rem);

	@include motion-safe {
		transition: opacity var(--transition-base),
		            transform var(--transition-base) var(--easing-decelerate);
	}

	&.is-open {
		opacity: 1;
		transform: translateY(0);
	}
}
```

When `prefers-reduced-motion: reduce` is set, the transition is never
declared — the browser applies the final state instantly on class
change.

## Duration tokens

| Token | Value | Use |
|---|---|---|
| `--transition-fast` | 150ms + standard easing | Hover, focus, small changes |
| `--transition-base` | 200ms + standard easing | Default UI motion |
| `--transition-slow` | 300ms + standard easing | Modal open/close, drawer |

## Easing tokens (curves only)

| Token | Curve | Use |
|---|---|---|
| `--easing-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | Default |
| `--easing-decelerate` | `cubic-bezier(0, 0, 0.2, 1)` | Enter animations |
| `--easing-accelerate` | `cubic-bezier(0.4, 0, 1, 1)` | Exit animations |
| `--easing-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Emphasis / success |
```

Commit:

```bash
git add docs/css/motion.md
git commit -m "docs(motion): add motion-safe + easing guide"
```
