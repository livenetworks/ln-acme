## Context

Every detail page in any consumer project needs a consistent header
pattern: title + breadcrumbs + action buttons. Today consumers
reinvent it per project. Approval workflows need a visual stepper.
Audit logs and version histories need a timeline.

Phase 7b ships all three as pure CSS — no JS, no interactivity. A
future phase (8 or later) can add `ln-timeline` JS if live-update
is needed.

## Pre-resolved ambiguities

### A. Page header structure

```html
<header data-ln-page-header>
	<nav aria-label="Breadcrumb">
		<ol>
			<li><a href="/">Home</a></li>
			<li><a href="/documents">Documents</a></li>
			<li aria-current="page">Quality Manual</li>
		</ol>
	</nav>

	<div>
		<h1>Quality Manual</h1>
		<p>Version 2.3 — Approved 2026-03-15</p>
	</div>

	<div>
		<button type="button">Edit</button>
		<button type="submit">Save</button>
	</div>
</header>
```

Three-slot layout:
1. `nav` (breadcrumbs) — top
2. `div` containing `h1` and optional `p` subtitle — middle
3. `div` containing action buttons — right of middle on wide screens,
   below middle on narrow

### B. Stepper structure

```html
<ol data-ln-stepper>
	<li data-ln-step="complete">
		<span data-ln-step-label>Draft</span>
	</li>
	<li data-ln-step="complete">
		<span data-ln-step-label>In Review</span>
	</li>
	<li data-ln-step="current" aria-current="step">
		<span data-ln-step-label>Approval</span>
	</li>
	<li data-ln-step="upcoming">
		<span data-ln-step-label>Published</span>
	</li>
</ol>
```

State attribute: `data-ln-step="complete|current|upcoming"`. CSS
selects on the attribute. No variant classes.

### C. Timeline structure

```html
<ol data-ln-timeline>
	<li>
		<time datetime="2026-04-14T09:32">Apr 14, 09:32</time>
		<h4>Document approved</h4>
		<p>Approved by Dalibor Sojic</p>
	</li>
	<li>
		<time datetime="2026-04-13T14:18">Apr 13, 14:18</time>
		<h4>Revision submitted</h4>
		<p>Draft saved with 3 changes</p>
	</li>
</ol>
```

Each `<li>` gets a bullet + vertical connector line via `::before` /
`::after`. No JS.

### Step 2 — Page header (Commit 1)

**Mixin** — `scss/config/mixins/_page-header.scss`:

```scss
@use 'spacing' as *;
@use 'display' as *;
@use 'typography' as *;
@use 'colors' as *;
@use 'borders' as *;

@mixin page-header {
	display: grid;
	grid-template-areas:
		'breadcrumbs'
		'title'
		'actions';
	@include gap(1rem);
	@include pb(1.5rem);
	@include mb(1.5rem);
	border-bottom: 1px solid hsl(var(--color-border));

	@container (min-width: 880px) {
		grid-template-areas:
			'breadcrumbs breadcrumbs'
			'title       actions';
		grid-template-columns: 1fr auto;
		align-items: end;
	}

	> nav {
		grid-area: breadcrumbs;
	}

	> div:has(> h1) {
		grid-area: title;

		h1 {
			@include typography(display-sm);
			margin: 0;
		}

		p {
			@include typography(body-md);
			color: hsl(var(--color-text-secondary));
			margin: 0.25rem 0 0;
		}
	}

	> div:has(> button),
	> div:has(> a) {
		grid-area: actions;
		@include flex-row;
		@include gap(0.5rem);

		@container (max-width: 879px) {
			justify-content: flex-start;
		}
	}
}
```

**Note:** `page-header` needs a container context to use the
`@container` query. If the parent of the header is already a
container (common in page layouts), it works out of the box.
Otherwise the header must get `@include container` applied externally.

**Component** — `scss/components/_page-header.scss`:

```scss
@use '../config/mixins' as *;

[data-ln-page-header] {
	@include page-header;
}
```

# Page header

File: `scss/config/mixins/_page-header.scss` + `scss/components/_page-header.scss`.

Standard page header pattern: breadcrumbs above, title + subtitle on
the left, action buttons on the right. Stacks vertically on narrow
containers.

## Structure

```html
<div class="container" style="container-type: inline-size;">
	<header data-ln-page-header>
		<nav aria-label="Breadcrumb">
			<ol>
				<li><a href="/">Home</a></li>
				<li aria-current="page">Quality Manual</li>
			</ol>
		</nav>
		<div>
			<h1>Quality Manual</h1>
			<p>Version 2.3 — Approved 2026-03-15</p>
		</div>
		<div>
			<button type="button">Edit</button>
			<button type="submit">Publish</button>
		</div>
	</header>
</div>
```

## Responsive

The header uses a container query at `880px`:
- Below 880px: stacks `breadcrumbs → title → actions` vertically
- At 880px+: breadcrumbs on top full-width, title-left + actions-right

The parent container must declare `container-type: inline-size` or
use `@include container` for the query to match.

## Slots

- `> nav` — breadcrumbs (top row)
- `> div` with `> h1` — title + optional `p` subtitle
- `> div` with `> button` or `> a` — action buttons

The `:has()` selector picks the correct div for each slot. Works in
all modern browsers (Chrome 105+, Safari 15.4+, Firefox 121+).

## Project selectors

```scss
#document-detail > header { @include page-header; }
```

		// Connector line to next step

			content: '';
			position: absolute;

# Stepper

File: `scss/config/mixins/_stepper.scss` + `scss/components/_stepper.scss`.

Linear progress indicator for multi-step workflows (approval flows,
wizards, onboarding).

## Structure

```html
<ol data-ln-stepper>
	<li data-ln-step="complete">
		<span data-ln-step-label>Draft</span>
	</li>
	<li data-ln-step="complete">
		<span data-ln-step-label>In Review</span>
	</li>
	<li data-ln-step="current" aria-current="step">
		<span data-ln-step-label>Approval</span>
	</li>
	<li data-ln-step="upcoming">
		<span data-ln-step-label>Published</span>
	</li>
</ol>
```

## States

| Attribute | Visual |
|---|---|
| `data-ln-step="complete"` | Filled primary circle, colored connector to next |
| `data-ln-step="current"` | Filled primary circle + halo ring |
| `data-ln-step="upcoming"` (default) | Neutral circle, neutral connector |

Number is auto-generated via CSS `counter()`. No explicit numbering
in HTML.

## Accessibility

- Use `<ol>` to convey sequence
- Mark the current step with `aria-current="step"` (in addition to
  `data-ln-step="current"`)
- Each step's accessible name is the `data-ln-step-label` text
```

# Timeline

File: `scss/config/mixins/_timeline.scss` + `scss/components/_timeline.scss`.

Vertical activity feed for audit logs, version history, comment
threads.

## Structure

```html
<ol data-ln-timeline>
	<li>
		<time datetime="2026-04-14T09:32">Apr 14, 09:32</time>
		<h4>Document approved</h4>
		<p>Approved by Dalibor Sojic</p>
	</li>

## Structure

- **`<time>`** — ISO datetime attribute + formatted text
- **`<h4>`** — event title (role `title-sm`)
- **`<p>`** — event description (role `body-sm`, secondary color)

## Rail

A vertical 2px rail runs the full height of the list. Bullets are
primary-coloured 12px circles with a 3px white (body) ring for
contrast.

## Project selectors

```scss

```

## Related

If you need interactive updates (new entries appearing, live filters),
consider a JS layer on top — but the baseline timeline is pure CSS.
