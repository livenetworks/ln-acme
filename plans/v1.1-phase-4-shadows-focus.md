**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 4
**Branch:** `v1.1/phase-4-shadows-focus`

**Unblocks:** Phase 6 (dark mode — cool-tinted shadows re-tune cleanly)

## Context

ln-acme v1.0 uses Tailwind-style neutral-black shadows:

```
--shadow-sm: 0 1px 3px 0 rgba(0, 0, 0, 0.1), 0 1px 2px -1px rgba(0, 0, 0, 0.1);
```

These work but look flat next to modern cool-tinted UI (shadows with a
tiny bit of blue hue read as "expensive" because they match the
ambient light model of daylight environments). The focus-ring mixin
produces a single 3px halo at 12% alpha — legible but perceptually
weak on backgrounds that share the primary hue.

This phase **replaces** shadow token values and **upgrades** the
focus-ring mixin. No component code changes; everything consuming the
tokens inherits the new look.

**Critical constraint preserved:** Button hover still only changes
color. No `translateY`, no hover shadow, no coloured click shadow.
`CLAUDE.md` § Button Architecture is explicit about this.

## Pre-resolved ambiguities

### A. Shadow hue

All shadows use `hsl(220 40% 15% / alpha)` as the base colour — matches
the `220` neutral hue from Phase 2, lightness floor at 15% to keep
shadows dark enough to read on white backgrounds.

### B. New tokens added vs. values changed

| Token | v1.0 | v1.1 action |
|---|---|---|
| `--shadow-none` | `none` | unchanged |
| `--shadow-xs` | `rgba(0,0,0,0.05)` single layer | replaced with cool dual-layer |
| `--shadow-sm` | dual rgba(0) | replaced |
| `--shadow-md` | dual rgba(0) | replaced |
| `--shadow-lg` | dual rgba(0) | replaced |
| `--shadow-xl` | dual rgba(0) | replaced |
| `--shadow-primary` | primary-tinted glow | replaced with updated value |
| `--shadow-2xl` | missing | **added** |
| `--shadow-inner` | missing | **added** (depressed state) |
| `--shadow-success` | missing | **added** |
| `--shadow-error` | missing | **added** |

### C. Focus-ring upgrade

Current `focus-ring` mixin:

```scss
@mixin focus-ring($color: var(--color-primary), $width: 3px) {
	box-shadow: 0 0 0 $width hsl($color / 0.12);
}
```

Problems: single layer; when the focused element sits on a background
that shares the primary hue, the ring is invisible. New version uses
**three stacked shadows**: a solid outer halo at the body-bg color
(acts as a separator), a saturated color ring, and a soft outer glow.

### D. Accessibility check

WCAG 2.1 § 1.4.11 requires 3:1 non-text contrast for focus indicators.
The saturated middle ring at 60% alpha on a standard `hsl(var(--color-primary))` = `232 75% 52%` easily clears 3:1
against white. Still verify in practice with a contrast checker.

### E. Commit strategy

Two commits:

1. `feat(tokens): refine shadow scale — cool-tinted, add 2xl/inner/success/error`
2. `feat(focus): upgrade focus-ring mixin to dual-ring halo`

### Step 2 — Refine shadows (Commit 1)

with:

```scss
	// -----------------------------------------------------------------------
	// Shadows — cool-tinted, dual-layer (v1.1)
	// -----------------------------------------------------------------------
	--shadow-none: none;

	--shadow-xs:
		0 1px 2px 0 hsl(220 40% 15% / 0.04);

	--shadow-sm:
		0 1px 3px 0 hsl(220 40% 15% / 0.08),
		0 1px 2px -1px hsl(220 40% 15% / 0.04);

	--shadow-md:
		0 4px 12px -2px hsl(220 40% 15% / 0.10),
		0 2px 4px -2px hsl(220 40% 15% / 0.06);

	--shadow-lg:
		0 12px 24px -6px hsl(220 40% 15% / 0.12),
		0 8px 12px -4px hsl(220 40% 15% / 0.08);

	--shadow-xl:
		0 24px 48px -12px hsl(220 40% 15% / 0.16),
		0 12px 24px -6px hsl(220 40% 15% / 0.10);

	--shadow-2xl:
		0 36px 72px -18px hsl(220 40% 15% / 0.20),
		0 18px 36px -9px hsl(220 40% 15% / 0.12);

		inset 0 2px 4px 0 hsl(220 40% 15% / 0.06);

	--shadow-error:   0 8px 24px -6px hsl(var(--color-error)   / 0.28);
```

### Step 3 — Upgrade focus-ring (Commit 2)

Edit `scss/config/mixins/_focus.scss`. Replace the `focus-ring` mixin
only (keep every other mixin in the file unchanged):

**Before:**

```scss
@mixin focus-ring($color: var(--color-primary), $width: 3px) {
	box-shadow: 0 0 0 $width hsl($color / 0.12);
}
```

**After:**

```scss
// Three-layer halo:
//   1. 2px ring at bg-primary colour — visual separator from the
//      element's own background
//   2. 2px ring at 60% of the colour — the main focus signal
//   3. 2px outer glow at 15% — soft halo outside the main ring
@mixin focus-ring($color: var(--color-primary)) {
	box-shadow:
		0 0 0 2px hsl(var(--color-bg-primary)),
		0 0 0 4px hsl($color / 0.6),
		0 0 0 6px hsl($color / 0.15);
	outline: none;
}
```

**Important:** the `$width` parameter is dropped — not backwards
compatible for callers that override width. Grep for non-default
callers first:

```bash
grep -rn 'focus-ring(' scss/ js/
```

Expected hits are all `@include focus-ring;` or `@include focus-ring(var(--color-error));`
(single-argument or zero-argument). If any caller passes a second
argument, halt and escalate — the mixin needs dual arity.

**Verify:**

Load `demo/admin/forms.html`. Tab through the form inputs. Each
focused input should show:
- A 2px white ring (separator) touching the element
- A 2px saturated blue ring outside that
- A 2px soft glow outside that (~6px total halo)

On an error field (`input.error:focus-visible`), the red variant
should look similarly layered. Test:

```scss
// scratch/_error-focus.scss
@use '../scss/config/mixins' as *;
.test:focus-visible { @include focus-ring(var(--color-error)); }
```

Compile, confirm the compiled CSS uses three `0 0 0` layers with
`var(--color-error)` in positions 2 and 3, and
`var(--color-bg-primary)` in position 1.

Delete `scratch/`.

**Accessibility:**

Open DevTools → Rendering → emulate `Forced colors` or use a browser
contrast-checking extension. Confirm the focus ring is visible on:
- White background (standard forms)
- `bg-secondary` background (settings panels)
- Primary-tinted background (selected rows) — the white separator
  ring guarantees visibility here

**Commit 2:**

### Step 4 — Document

Append to `docs/css/tokens.md` in the Shadows section:

```markdown
### Shadow scale (v1.1)

All shadows use `hsl(220 40% 15% / alpha)` as the base — gives a faint
cool tint that reads as modern/expensive on daylight backgrounds.

| Token | Layers | Use |
|---|---|---|
| `--shadow-none` | — | Reset |
| `--shadow-xs` | 1 | Hairline lift (subtle cards) |
| `--shadow-sm` | 2 | Cards, dropdowns |
| `--shadow-md` | 2 | Floating panels, popovers |
| `--shadow-lg` | 2 | Modals, toast |
| `--shadow-xl` | 2 | Full-screen overlays |
| `--shadow-2xl` | 2 | Hero elevation |
| `--shadow-inner` | 1 | Depressed state (active, selected) |
| `--shadow-primary` | 1 | Primary-tinted glow (CTA emphasis, focus halo) |
| `--shadow-success` | 1 | Success-tinted glow |
| `--shadow-error` | 1 | Error-tinted glow |

**Do not use hover shadows on buttons.** Buttons change colour only
(`CLAUDE.md` § Button Architecture).
```

Append to `docs/css/tokens.md` a new "Focus" subsection:

```markdown
### Focus ring (v1.1)

`@include focus-ring($color: var(--color-primary))` produces a
three-layer halo:

1. 2px inner ring in `--color-bg-primary` — visual separator
2. 2px middle ring in `$color` at 60% alpha — main signal
3. 2px outer glow in `$color` at 15% alpha — soft halo

Error variant: `@include focus-ring(var(--color-error))`.

The v1.0 `$width` argument has been removed — the three-layer halo is
always 6px total. Projects that need a different thickness can use
`focus-combination`, `focus-inset-shadow`, or write a custom mixin.
```
