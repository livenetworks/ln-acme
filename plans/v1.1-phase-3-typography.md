# Plan: v1.1 Phase 3 — Typography Re-hierarchy

**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 3
**Branch:** `v1.1/phase-3-typography`
**Depends on:** Phase 1 (token additions) and Phase 2 (neutral scale)
**Unblocks:** Phase 7 (CSS components batch — page-header, prose, stat-card

---

## Context

ln-acme v1.0 typography ships 6 `text-*` mixins (xs → 2xl) that set
`font-size` + hardcoded `line-height`. There are no semantic roles —
every consumer decides independently whether `h1` is `text-2xl` or a
bespoke size, whether body copy is `text-base` or `text-sm`. DocuFlow
already overrides every single typography token in
`resources/scss/_layout.scss`. That's the symptom.

This phase **adds** a semantic layer on top of the existing primitives.
Primitive `text-*` mixins stay — they are the low-level building blocks.
The new `typography($role)` mixin calls them by role name.

## Pre-resolved ambiguities

### A. Coexistence with existing `text-*` mixins

Do NOT touch `_typography.scss` mixin definitions. `text-xs` through
`text-2xl` remain byte-identical. The new `typography()` mixin is the
semantic layer; consumers migrate at their own pace.

### B. Role token values

Hand-tuned modular scale, 1.25 ratio approximately:

```
display-lg   3.75rem  / lh 1.1
display-md   3rem     / lh 1.1
display-sm   2.25rem  / lh 1.15
heading-lg   1.875rem / lh 1.2
heading-md   1.5rem   / lh 1.25
heading-sm   1.25rem  / lh 1.3
title-md     1.125rem / lh 1.4
body-lg      1.125rem / lh 1.6
body-md      1rem     / lh 1.6
body-sm      0.875rem / lh 1.5
label-md     0.875rem / lh 1.4
label-sm     0.75rem  / lh 1.4
caption      0.75rem  / lh 1.4
```

### C. Heading mapping

`scss/base/_typography.scss` currently maps headings as follows:

| Tag | Current size | New role |
|---|---|---|
| `h1` | 2.25rem | `display-sm` (2.25rem — exact match) |
| `h2` | `text-2xl` = 1.5rem | `heading-md` (1.5rem — exact match) |
| `h3` | `text-xl` = 1.25rem | `heading-sm` (1.25rem — exact match) |
| `h4` | `text-lg` = 1.125rem | `title-md` (1.125rem — exact match) |
| `h5` | `text-base` = 1rem | `body-lg` (1.125rem) ❌ drift |
| `h6` | `text-sm` = 0.875rem | `label-md` (0.875rem — exact match) |

**Drift resolution for `h5`:** the current 1rem is correct for `h5`
and matches a `title-sm` role that isn't in the scale. Either:

(a) add `--text-title-sm: 1rem; --lh-title-sm: 1.4;` and keep `h5`
visually identical, or

(b) accept the 1rem → 1.125rem drift (12.5% larger) as an intentional
normalization.

**Decision:** option (a). Headings are load-bearing; changing `h5`
size by 12.5% is a visible shift every project would notice. Add
`title-sm` to the scale.

**Final scale:**

```
display-lg   3.75rem   / 1.1
display-md   3rem      / 1.1
display-sm   2.25rem   / 1.15
heading-lg   1.875rem  / 1.2
heading-md   1.5rem    / 1.25
heading-sm   1.25rem   / 1.3
title-md     1.125rem  / 1.4
title-sm     1rem      / 1.4
body-lg      1.125rem  / 1.6
body-md      1rem      / 1.6
body-sm      0.875rem  / 1.5
label-md     0.875rem  / 1.4
label-sm     0.75rem   / 1.4
caption      0.75rem   / 1.4
```

### D. Heading line-heights

Current `_typography.scss` hardcodes `line-height: 1.25` globally and
then overrides per-tag (h1: 2.5rem, h3: 1.1, h4: 1.1, h5: 1.1, h6: 1.1).
The new role tokens replace those hardcodes.

### E. Tabular numerals

Add `font-variant-numeric: tabular-nums` to the `table-base` mixin so
numeric columns align by default. Surgical single-line addition.

```scss
// Semantic role mixin — v1.1.
// Usage: @include typography(display-lg) / typography(body-md) / etc.
@mixin typography($role) {
	font-size:   var(--text-#{$role});
	line-height: var(--lh-#{$role});
}

@mixin tracking-tight  { letter-spacing: var(--tracking-tight); }
@mixin tracking-normal { letter-spacing: var(--tracking-normal); }
```

```markdown
# Typography

Files:
- `scss/config/_tokens.scss` — role and tracking tokens
- `scss/config/mixins/_typography.scss` — `text-*` primitives + `typography($role)` mixin
- `scss/base/_typography.scss` — default heading and paragraph styling

## Two layers

ln-acme typography has two layers:

1. **Primitives** — `text-xs`, `text-sm`, `text-base`, `text-lg`,
   `text-xl`, `text-2xl`. Plain font-size + line-height mixins. Use
   when the intent is literally "make this 14px".

2. **Semantic roles (v1.1)** — `typography(<role>)`. Use when the
   intent is "this is a page title" or "this is a form label". The
   role determines the size; the size is a design decision, not a
   caller decision.

**Projects should prefer roles.** Primitives stay as escape hatches.

## Role selection guide

| Context | Role |
|---|---|
| Hero / marketing headline | `display-lg`, `display-md` |
| Page title (`h1`) | `display-sm` |
| Section headline (`h2`) | `heading-md` |
| Subsection (`h3`) | `heading-sm` |
| Card title (`h4`) | `title-md` |
| Strong body lead (`h5`) | `title-sm` |
| Form label (`h6`, `<label>`) | `label-md` |
| Standard body copy | `body-md` |
| Helper text / hints | `body-sm` |
| Meta / timestamps | `caption` |

## Usage

```scss
@use 'ln-acme/scss/config/mixins' as *;

.document-title { @include typography(display-sm); @include font-bold; }
.meta           { @include typography(caption); @include text-muted; }
.form-field label { @include typography(label-md); }
```

## Letter spacing

| Token | Value | Use |
|---|---|---|
| `--tracking-tight` | -0.025em | Display / heading sizes |
| `--tracking-normal` | 0 | Body |
| `--tracking-wide` | 0.025em | Uppercase labels, badges |

## Tabular numerals

As of v1.1, `table-base` sets `font-variant-numeric: tabular-nums` so
numeric columns align vertically by default. Apply manually elsewhere:

```scss
.kpi-value { font-variant-numeric: tabular-nums; }
```

## Defaults

`scss/base/_typography.scss` maps `h1`–`h6` to role tokens so
unstyled HTML is correct out of the box. Consuming projects can
override per-page with their own `@include typography(...)` calls.
