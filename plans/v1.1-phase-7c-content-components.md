# Plan: v1.1 Phase 7c — Content Display

**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 7

**Depends on:** Phases 1, 2, 3, 4 complete.

**Components in this batch:**
1. `stat-card` / `kpi-card` — value + label + trend slot
2. `prose` — rich-text content (TipTap output, long-form)
3. `kbd` — keyboard key representation inline

Batch focus: displaying content richly. Dashboards, documentation,
rich-text output.

---

## Context

Three content-display components. Each is pure CSS, zero JS.

1. **stat-card** — dashboards need a consistent KPI pattern. Spec
   exists in claude-skills as `kpi-card.md`. Phase 7c pulls it in.
2. **prose** — TipTap (DocuFlow's editor) outputs semantic HTML. That
   HTML needs consistent styling in the document viewer. `@include
   prose` is the answer.
3. **kbd** — `<kbd>` element needs keycap-looking styling. 20 lines
   of CSS, but high-value everywhere keyboard shortcuts are documented.

## Pre-resolved ambiguities

### A. stat-card structure

```html
<article data-ln-stat-card>
	<p data-ln-stat-label>Total documents</p>
	<p data-ln-stat-value>1,247</p>
	<p data-ln-stat-trend="up">
		<svg class="ln-icon" aria-hidden="true"><use href="#ln-arrow-up"></use></svg>
		12% from last month
	</p>
</article>
```

`data-ln-stat-trend` value: `up` (success) / `down` (error) /
`neutral` (muted).

Value uses `tabular-nums` (Phase 3 made table-base do this by
default; stat-card does it explicitly for non-table usage).

### B. prose structure

```html
<article data-ln-prose>
	<!-- arbitrary TipTap output: h1-h6, p, ul, ol, blockquote, pre, table, img -->
</article>
```

`@include prose` applies styling to descendants via selector. No
per-element classes expected. TipTap outputs semantic HTML.

### C. kbd structure

```html
<p>Press <kbd>Ctrl</kbd> + <kbd>K</kbd> to open the command palette.</p>
```

All `<kbd>` elements get keycap styling by default. No class, no
attribute.

File: `scss/config/mixins/_stat-card.scss` + `scss/components/_stat-card.scss`.

## Trend variants

| Attribute | Color |
|---|---|
| `data-ln-stat-trend="up"` | Success (green) |
| `data-ln-stat-trend="down"` | Error (red) |
| `data-ln-stat-trend="neutral"` | Muted grey |

## Layout

Stat cards are typically displayed in a grid. Use container queries
to lay them out:

```scss

	// card styling inherited from component
}

	@include container(kpigrid);
	display: grid;
	grid-template-columns: 1fr;
	@include gap(1rem);

	@container kpigrid (min-width: $cq-compact) {
		grid-template-columns: repeat(2, 1fr);
	}
	@container kpigrid (min-width: $cq-medium) {
		grid-template-columns: repeat(4, 1fr);
	}
}
```

## Project selectors

# Prose

File: `scss/config/mixins/_prose.scss` + `scss/components/_prose.scss`.

Semantic rich-text styling for content that comes from rich editors
(TipTap, CKEditor, Markdown renderers) or long-form article content.

## Structure

```html
<article data-ln-prose>
	<!-- arbitrary semantic HTML -->
	<h2>Introduction</h2>
	<p>Lorem ipsum...</p>
	<ul>
		<li>Point one</li>
	</ul>
</article>
```

## What it styles

- Headings (`h1`–`h4`) with role tokens + spacing
- Paragraphs (`body-md`)
- Links (primary colour, underline)
- Lists (`ul`, `ol`) with proper nesting
- Blockquotes (primary left border)
- Inline code + code blocks (mono, bg-secondary)
- Images (max-width 100%)
- Figures with captions
- Tables (unstyled wrapper — use `@include table-base` for full
  styling)
- Horizontal rules

## Max width

`max-width: 65ch` (via `--max-w-prose`). Optimized for reading.
Override per-consumer if needed.

## Project selectors

```scss
#document-viewer { @include prose; }
#help-article    { @include prose; }
```

## TipTap output

TipTap outputs this subset of semantic HTML by default. `@include
prose` is the recommended wrapper styling for any TipTap-rendered
content.

# Kbd

File: `scss/config/mixins/_kbd.scss` — applied to `kbd` in `scss/base/_typography.scss`.

- Monospace font
- Neutral background + border
- Thicker bottom border (keycap illusion)

For cross-platform docs, use `<kbd>` with the canonical key name:

```html

```
