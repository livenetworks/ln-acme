# Plan: v1.1 Phase 2 — Neutral Color Scale + Semantic Remap

**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 2
**Branch:** `v1.1/phase-2-neutral-scale`
**Depends on:** Phase 1 complete and merged
**Unblocks:** Phase 3, Phase 4, Phase 6 (dark mode)

---

## Context

ln-acme v1.0 has implicit greys scattered across `--color-text-*`,
`--color-bg-*`, `--color-border*`, and `--color-table-*`. There is no
`--color-neutral-50` → `--color-neutral-900` chain. Dark mode is
currently impossible without this chain: every semantic token would
need to be flipped individually, and the inter-token contrast
relationships would drift.

This phase **adds** a neutral scale with HSL values chosen so that
remapping the existing semantic tokens to `var(--color-neutral-*)`
produces **identical pixels**. It is not a pure addition — the
semantic tokens now reference the scale instead of holding bare
values. Visual regression budget matters.

## Pre-resolved ambiguities

### A. Scale values must match current HSL exactly

Current bare HSL triplets in `_tokens.scss`:

| Semantic token | Current HSL triplet |
|---|---|
| `--color-text-primary` | `221 39% 11%` |
| `--color-text-secondary` | `220 9% 46%` |
| `--color-text-muted` | `218 11% 65%` |
| `--color-bg-primary` | `0 0% 100%` |
| `--color-bg-secondary` | `220 20% 97%` |
| `--color-bg-body` | `220 27% 96%` |
| `--color-border` | `220 13% 91%` |
| `--color-border-light` | `220 14% 95%` |
| `--color-table-header-bg` | `220 14% 91%` |
| `--color-table-header-text` | `221 39% 11%` |
| `--color-table-section-bg` | `220 25% 93%` |

Problem: these values do not form a clean 50→900 scale. The hues drift
(218/220/221), the lightness steps are irregular (95/96/97), and there
are duplicates (`--color-table-header-bg` = `--color-border` = `220
14% 91%` vs `220 13% 91%`).

**Decision:** Build a clean 50→900 scale around the DocuFlow-approved
royal-blue neutral hue of `220`, then remap semantic tokens to the
nearest scale step. Document the tiny drifts (≤1% lightness) as
intentional normalization. Anything larger must be reviewed.

**Approved scale:**

```
--color-neutral-50:  220 20% 98%;    // was "almost white" (~97.5%)
--color-neutral-100: 220 16% 96%;    // was bg-body (96%) — matches
--color-neutral-150: 220 14% 93%;    // NEW tier — was table-section-bg
--color-neutral-200: 220 13% 91%;    // was border/table-header-bg (91%) — exact
--color-neutral-300: 220 13% 83%;
--color-neutral-400: 218 11% 65%;    // was text-muted — exact
--color-neutral-500: 220  9% 46%;    // was text-secondary — exact
--color-neutral-600: 220 11% 34%;
--color-neutral-700: 220 14% 24%;
--color-neutral-800: 220 20% 15%;
--color-neutral-900: 221 39% 11%;    // was text-primary — exact
```

**Drift analysis:**

| Semantic | Before | Remap | After | Delta |
|---|---|---|---|---|
| `bg-primary` | `0 0% 100%` | `var(--color-white)` | `0 0% 100%` | 0 |
| `bg-body` | `220 27% 96%` | `var(--color-neutral-100)` | `220 16% 96%` | saturation −11% (imperceptible on neutrals) |
| `bg-secondary` | `220 20% 97%` | `var(--color-neutral-50)` | `220 20% 98%` | +1% lightness |
| `text-primary` | `221 39% 11%` | `var(--color-neutral-900)` | `221 39% 11%` | 0 |
| `text-secondary` | `220 9% 46%` | `var(--color-neutral-500)` | `220 9% 46%` | 0 |
| `text-muted` | `218 11% 65%` | `var(--color-neutral-400)` | `218 11% 65%` | 0 |
| `border` | `220 13% 91%` | `var(--color-neutral-200)` | `220 13% 91%` | 0 |
| `border-light` | `220 14% 95%` | `var(--color-neutral-100)` | `220 16% 96%` | +1% lightness, +2% sat |
| `table-header-bg` | `220 14% 91%` | `var(--color-neutral-200)` | `220 13% 91%` | −1% sat (invisible) |
| `table-section-bg` | `220 25% 93%` | `var(--color-neutral-150)` | `220 14% 93%` | saturation −11% |

All drifts are ≤1% lightness OR saturation-only on near-neutral greys
where saturation is visually indistinguishable. Within the 1%
visual-regression budget.

**If the user cannot approve this drift:** fall back to keeping the
current semantic tokens as bare HSL but adding the neutral scale
alongside, un-referenced. Phase 6 dark mode then has to flip each
semantic token individually. Less clean but zero-drift.

### B. `--color-neutral-150`

Non-standard but necessary — the current `--color-table-section-bg`
at 93% sits between the canonical 100 and 200 tiers, and eliminating
it would force a visible shift on every striped table. Document it as
a deliberate intermediate step.

### C. Commit strategy

Two commits:

1. `feat(tokens): add neutral color scale --color-neutral-50 through 900`
2. `refactor(tokens): remap semantic colors to neutral scale`

Separates addition from remap, so a bisect can pin regressions to the
remap commit specifically.

### Step 3 — Remap semantic tokens (Commit 2)

In the same file, replace the existing `// Colors — Text`, `// Colors — Backgrounds`,
`// Colors — Borders`, and `// Colors — Table-specific` blocks with:

```scss
	// -----------------------------------------------------------------------
	// Colors — Text (reference neutral scale)
	// -----------------------------------------------------------------------
	--color-text-primary:   var(--color-neutral-900);
	--color-text-secondary: var(--color-neutral-500);
	--color-text-muted:     var(--color-neutral-400);

	// -----------------------------------------------------------------------
	// Colors — Backgrounds
	// -----------------------------------------------------------------------
	--color-white:         0 0% 100%;
	--color-bg-primary:    var(--color-white);
	--color-bg-secondary:  var(--color-neutral-50);
	--color-bg-body:       var(--color-neutral-100);
	--color-bg-error:      0 86% 97%;

	// -----------------------------------------------------------------------
	// Colors — Borders
	// -----------------------------------------------------------------------
	--color-border:       var(--color-neutral-200);
	--color-border-light: var(--color-neutral-100);

	// -----------------------------------------------------------------------
	// Colors — Table-specific
	// -----------------------------------------------------------------------
	--color-table-header-bg:   var(--color-neutral-200);
	--color-table-header-text: var(--color-neutral-900);
	--color-table-section-bg:  var(--color-neutral-150);
```

`hsl(var(--color-neutral-900))` which is `hsl(221 39% 11%)`. CSS
custom-property indirection through `hsl()` works — verified in browser

### Neutral scale (v1.1)

The canonical grey layer. All semantic colors (text, background,
border, table) reference this scale. Dark mode inverts the scale

|---|---|---|
