# Plan: v1.1 Phase 9 — Density System

**Source roadmap:** `plans/design-system-v1.1-roadmap.md` § Phase 9
**Branch:** `v1.1/phase-9-density`
**Depends on:** Phase 1 (spacing tokens), Phase 3 (typography roles)

---

## Context

DocuFlow and other ISO/admin apps are data-dense. The default
ln-acme spacing scale works for general UI but gets too airy on
tables with 50+ rows or settings panels with 20+ fields.

Today the only way to compact a table is to manually override
padding everywhere. Phase 9 adds a `[data-density="compact"]`
scope that scales spacing and typography tokens inside the marked
subtree.

DocuFlow-specific: the current `--text-xs: 0.6875rem` (11px)
override in DocuFlow is a symptom of this gap. After Phase 9,
DocuFlow can drop that global override and apply `data-density="compact"`
only to data cells.

## Pre-resolved ambiguities

### A. Scope granularity

Density applies to a **subtree**, not globally. Apply
`data-density="compact"` to the root of the section that should be
dense:

```html
<section data-density="compact">
	<!-- dense table, compact form, whatever -->
</section>
```

Outside that section, spacing/typography returns to default.

### B. What density changes

**Scales:** spacing (all steps), body typography (body-md, body-sm,
label-md, label-sm).

**Does NOT scale:** display sizes, headings, focus rings, border
radii, shadows, colors. Compact mode is about fitting more content,
not changing the design language.

### C. Scale factor

Approximately 0.75× on spacing and body text:

```
Normal → Compact

--spacing-xs:  0.25rem   → 0.1875rem
--spacing-sm:  0.5rem    → 0.375rem
--spacing-md:  1rem      → 0.75rem
--spacing-lg:  1.5rem    → 1.125rem
--spacing-xl:  2rem      → 1.5rem
--spacing-2xl: 3rem      → 2.25rem

--text-body-md: 1rem     → 0.875rem
--text-body-sm: 0.875rem → 0.8125rem
--text-label-md: 0.875rem → 0.8125rem
--text-label-sm: 0.75rem → 0.6875rem
```

### D. Compact floor

The smallest text in compact mode is `label-sm` at `0.6875rem` (11px).
This is below the WCAG recommended 12px floor. **Use only for data
cells, never for reading content.** Document this clearly.

### E. Multiple density levels

Only `normal` and `compact` for v1.1. No `cozy` / `spacious` /
`comfortable`. YAGNI until proven.

### F. Table retrofit

`table-base` mixin should respect density automatically — no consumer
work needed. The mixin already references `--spacing-*` for `td`
padding; since those tokens are overridden under the density scope,
nothing to change in `_table.scss`.

**Verify by reading `_table.scss` and confirming all padding uses
`var(--spacing-*)` or the equivalent `@include py/px(rem)` with
literal values.** If literal rems are used, patch them to reference
tokens (this was Phase 3's implicit work but might have been missed).

### G. Commit strategy

Two commits:

1. `feat(density): add [data-density="compact"] token overrides`
2. `docs(density): add density guide`

**Step 2b (conditional)** — replace literal rems with token

```scss

### Step 3 — Add density overrides (Commit 1)

Append to `scss/config/_tokens.scss`, at the very end of the file
(after the closing `}` of `:root`):

```scss
// ==========================================================================
// Density — [data-density="compact"] scoped overrides
// ==========================================================================
// Applies a compact spacing + typography scale to any subtree.
//
// Usage:
//   <section data-density="compact">
//     <!-- dense table, compact form -->
//   </section>
//
// See docs/css/density.md for the full guide.

[data-density="compact"] {
	// Spacing — ~0.75×
	--spacing-xs:  0.1875rem;
	--spacing-sm:  0.375rem;
	--spacing-md:  0.75rem;
	--spacing-lg:  1.125rem;
	--spacing-xl:  1.5rem;
	--spacing-2xl: 2.25rem;

	// Body typography — one step smaller
	--text-body-md:  0.875rem;
	--text-body-sm:  0.8125rem;
	--text-label-md: 0.8125rem;
	--text-label-sm: 0.6875rem;
}
```

**Verify:**

```bash
npm run build
grep -c 'data-density="compact"' dist/ln-acme.css
```

Expected: ≥1.

### Step 4 — Docs + demo (Commit 2)

Create `docs/css/density.md`:

```markdown
# Density

File: `scss/config/_tokens.scss` (`[data-density="compact"]` block).

ln-acme ships with a default spacing and typography scale tuned for
comfortable reading. Data-dense views (long tables, settings forms
with 20+ rows) benefit from a compact scale.

## Activation

Wrap the section that should be dense in a container with
`data-density="compact"`:

```html
<section data-density="compact">
	<table>
		<!-- dense table -->
	</table>
</section>
```

Outside the wrapper, spacing and typography return to default.

## What scales

| Token | Normal | Compact |
|---|---|---|
| `--spacing-xs` | 0.25rem (4px) | 0.1875rem (3px) |
| `--spacing-sm` | 0.5rem (8px) | 0.375rem (6px) |
| `--spacing-md` | 1rem (16px) | 0.75rem (12px) |
| `--spacing-lg` | 1.5rem (24px) | 1.125rem (18px) |
| `--spacing-xl` | 2rem (32px) | 1.5rem (24px) |
| `--spacing-2xl` | 3rem (48px) | 2.25rem (36px) |
| `--text-body-md` | 1rem (16px) | 0.875rem (14px) |
| `--text-body-sm` | 0.875rem (14px) | 0.8125rem (13px) |
| `--text-label-md` | 0.875rem (14px) | 0.8125rem (13px) |
| `--text-label-sm` | 0.75rem (12px) | 0.6875rem (11px) |

## What does NOT scale

- Display sizes (`display-*`) — hero text stays hero size
- Headings (`heading-*`) — h1-h4 stay hero size
- Focus rings, borders, shadows, radii — structure preserved
- Colors — identical
- Display typography (subtitles, stat values)

## When to use

- **Yes:** data tables with 50+ rows, settings forms with 20+ fields,
  admin lists, dashboards with many KPI tiles
- **No:** long-form reading content, marketing pages, onboarding
  flows

## Accessibility warning

Compact `--text-label-sm` is **11px** — below the WCAG recommended
12px floor for body text. Use compact labels only for secondary
metadata (row timestamps, status pills), never for primary
readable content.

Compact `--text-body-md` is 14px — still within WCAG AAA
recommendations for body text.

## Nesting

Density scopes compose. A compact section inside a compact section is
identical to a single compact section — tokens are already overridden
at the outer layer.

```html
<section data-density="compact">
	<!-- outer: everything compact -->
	<div>
		<table>...</table>
	</div>
</section>
```

## Custom density levels

Not currently supported. Only `compact` exists beyond the default.
Custom levels can be added by extending `_tokens.scss` with a new
scope — but YAGNI applies. Prove the need before adding.
```
